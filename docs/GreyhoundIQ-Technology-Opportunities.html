Adopt Now

The shortlist

Ten technologies worth committing to, each mapped to the GreyhoundIQ feature it serves. All are open-source or already in the stack, and all keep Australian data on infrastructure you control.

Mem0 + Supabase pgvector

Open-source (Apache-2.0) per-user memory layer; auto-extracts facts/preferences; pluggable vector store.

GreyhoundIQ → The drop-in answer to per-user AI agent memory — store each punter's tracks, dogs followed, staking style and each breeder's priority markers in the Supabase Postgres you already run. No new datastore. Top pick in 4 of 6 corpus batches.

Hermes + Honcho dialectic memory

Honcho models the user (beliefs + confidence + contradiction tracking), native to the Hermes runtime already in the stack.

GreyhoundIQ → Richer than flat memory — an agent that notices “you said you avoid maidens but backed three last week.” Validates the existing Hermes-on-MiniMax-M3 choice. Licensing gotcha: Honcho is AGPL-3.0 — isolate behind a service boundary, or use Mem0 instead.

pgvector as the one store

Vectors live beside relational data in Postgres; ~45ms p99 on 500k vectors in the corpus test.

GreyhoundIQ → Keep prediction features, form-search embeddings, breeding vectors and agent memory in one Supabase Postgres — a prediction query joins dog stats + semantic similarity in a single SQL call. Qdrant is the documented escape hatch only if pgvector strains.

Agent security: scan + sandbox + gate

Skill Spectre / NVIDIA SkillSpector scan skills for 14 attack types; Docker default-deny sandbox; tiered human approval gates.

GreyhoundIQ → Non-negotiable. Hermes runs skills with live Topaz / Betfair / Supabase credentials — a poisoned skill is host takeover (the real CVE-2026-25253 OpenClaw RCE + 341 malicious ClawHub skills). Scan every skill, sandbox execution, gate all DB mutations.

Prompt-eval pipeline (race results = ground truth)

Git-versioned prompts, auto-evaluated on every change against a held-out set, with regression gates that block >5% accuracy drops.

GreyhoundIQ → Greyhound racing gives free, automatic ground truth — past race results. Score win-probability accuracy & calibration on every model/prompt change and block silent regressions in a product people bet on. The single cheapest quality safeguard.

TimesFM + Hugging Face AutoTrain

Pretrained time-series foundation model (zero-training inference) + no-code training on the HF hub.

GreyhoundIQ → TimesFM forecasts form trajectories and gives a cheap baseline to benchmark the ML win-probability model; AutoTrain is the realistic path to an actual prediction model on Topaz history without an ML hire. Self-host weights to avoid Inference-API data egress.

n8n (self-hosted) ingestion

Self-hostable workflow automation with native AI nodes; no per-task fee; data stays on the box.

GreyhoundIQ → Orchestrate the Topaz / Betfair ingestion + enrichment into Supabase — schedule polls, classify scratchings, route on parsed JSON, fire DNA recessive-risk alerts. Glue, not system-of-record: Prisma/Supabase stays authoritative.

Firecrawl via MCP (gap-fill)

Web-context API returning clean LLM-ready Markdown/JSON; self-hostable; exposed as an MCP server.

GreyhoundIQ → Ingestion fallback where Topaz/Betfair don't reach — stewards' reports, scratchings, track-condition notes, trainer news. Adopt deliberately: self-host to keep data on Hetzner, and check scraping ToS for each source.

Cognee knowledge graph

Auto-extracts entities + relationships into a queryable graph for multi-hop “graph RAG”; supports pgvector + Anthropic.

GreyhoundIQ → Built for DNA / breeding — model sire/dam/offspring/kennel/recessive-carrier as graph nodes to answer what vector search can't: “which available studs share no recessive-risk carriers with this bitch's line?” Pairs with Testmate + the inbreeding-coefficient feature.

LocalAI / Whisper + Piper / Voicebox

OpenAI-compatible local inference + on-box STT (Whisper) and TTS (Piper/Voicebox).

GreyhoundIQ → Two wins at once: a data-sovereignty path (run embeddings + a fallback LLM on Hetzner, base-URL swap, AU data never leaves) and the voice-messaging feature (transcribe voice notes, auto-captions, narrated form guides) without shipping user audio to a third party.

Stack Validation

What the corpus confirmed

Several articles independently checked the choices already baked into GreyhoundIQ — a useful signal that the foundation is sound before layering anything new on top.

MiniMax M3 leads open-weight

Multiple model-comparison pieces put M3 at the top of the open-weight tier (59% SWE-bench Pro, 1M context, MSA sparse attention). The Hermes-on-M3 plan is the right engine — just budget the real tiered price, not the launch promo.

Hermes is the right runtime

The Hermes-vs-OpenClaw analyses favour Hermes for per-user learning via Honcho, and the deployment-patterns piece names “Hermes on a Hetzner VPS” as a first-class pattern — exactly the GreyhoundIQ deployment.

Supabase + pgvector is enough

The vector-DB comparisons conclude pgvector-in-Postgres beats bolting on a sixth service at GreyhoundIQ's scale — one store for relational data, embeddings, auth and realtime.

Self-host keeps data sovereign

The Australian governance pieces (Privacy Act, CPS 230, anti-“AI-washing”) reinforce keeping inference and PII on AU infrastructure — which the Hetzner + LocalAI path makes achievable.

Filtered Out

What to skip or watch — and why

Being explicit about what not to chase is half the value of reading 350 articles. These recur in the news cycle but don't fit GreyhoundIQ today.

Skip / watch list

Skip: trillion-parameter RL (PrimeRL) — the prediction problem is tabular ML, not LLM RL; OSINT dashboards, image-to-3D, game-engine MCPs — no domain surface; datacentre serving architectures — single-VPS scale. Watch only: 12M-context models (SubQ) — closed weights + lock-in, and 1M already covers form+pedigree; exotic compute (Bolt Zeus, photonic RPU) — tabular ML too light to justify. Gate carefully: DeepSeek-V4-Flash is the most tempting cost lever but its Chinese hosting is a hard AU data-sovereignty blocker for user PII — pilot on non-sensitive batch work only, behind a privacy review.
